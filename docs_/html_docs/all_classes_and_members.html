core.api package

Package provides the core ApiClient class for interacting with the web API.

The ApiClient class handles tasks like:

  • Sending API requests and receiving responses.

  • Managing authentication tokens.

  • Handling errors and exceptions.

Use this class for all interactions with the backend API of your application.

core.api.api_client module

APIClient module as class provides a central interface for interacting with web APIs.

This class offers methods to:

  • Send API requests using various HTTP methods (GET, POST, PUT, PATCH, DELETE).

  • Manage base URL and request session.

  • Handle request headers, JSON bodies, and query parameters.

  • Log API requests and responses.

Method to execute API requests with parameters for method, endpoint, headers, JSON body, and query parameters.

Parameters:

  • method (str) – HTTP method to use for the request (e.g., ‘GET’, ‘POST’).

  • endpoint (str) – API endpoint to send the request to (e.g., ‘/resource’).

  • headers (Optional[Dict[str, str]]) – Optional dictionary of headers to include in the request (e.g., authorization tokens).

  • json (Optional[Dict[str, Any]]) – Optional dictionary containing the JSON data to include in the request body (for POST, PUT, etc.).

  • params (Optional[Dict[str, Any]]) – Optional dictionary containing query parameters to include in the request URL.

config package

Package contains essential configuration files and utility modules used for managing database settings, logging configuration, code linting rules, and test execution settings. These files help streamline the development and testing process in a Python-based test automation framework.

tests package

Package includes test for

  • back-end API

tests.api_tests package

Package includes tests for:

  • the app business core functionality by executing back-end and front-end API testing.

  • some basic API performance check

  • basis JSON validation of most important API requests
